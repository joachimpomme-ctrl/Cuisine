# Exclusions `cap_techniques_v1`

## Exclusions fermes

- fiches recettes complètes ;
- quantités détaillées de recettes ;
- fiches techniques CAP de plats ;
- listes de matériel exhaustives ;
- règles de dressage décoratif ;
- consignes d'envoi ou de service d'examen ;
- pâtisserie recette par recette ;
- consignes locales de lycée non universelles ;
- temps et températures trop liés à une fiche scolaire donnée.

## Exclusions partielles

### Cuissons déjà couvertes

À ne pas répliquer sauf apport réellement nouveau :

- rôtir
- frire
- pocher
- vapeur
- déglacer
- réduire

### Assaisonnement déjà couvert

À exclure presque totalement :

- assaisonnements de sauces comme recettes ;
- ratios d'assaisonnement ;
- corrections de goût spécifiques à une recette.

### Rattrapage

À ne pas recréer :

- gestion du trop salé
- trop acide
- trop sucré
- trop épicé
- viande sèche
- légumes trop cuits

## Contenus trop professionnels pour la cuisine domestique

- bridages spécialisés non utiles à la plupart des usages domestiques ;
- découpes avancées orientées restauration ;
- fiches de rendement ou de dressage ;
- opérations de production collective.

## Pourquoi ces exclusions

Le projet vise une base de règles :

- atomiques ;
- actionnables ;
- transférables ;
- utiles à un agent IA domestique.

Tout ce qui ressemble à :

- une recette complète ;
- un protocole d'établissement ;
- un matériel de laboratoire ;
- une exigence d'examen ;

doit rester hors lot.
